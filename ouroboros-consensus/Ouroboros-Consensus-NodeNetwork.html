data ProtocolHandlers m peer blk Source #

The collection of all the mini-protocol handlers provided by the consensus layer.

These are used to construct the server (responder) and client (initiator) sides of the node-to-node protocol bundle, and the node-to-client protocol bundle.

data NetworkApplication m peer localPeer bytesCS bytesBF bytesTX bytesLCS bytesLTX bytesLSQ a Source #

Consensus provides a chains sync, block fetch applications. This data type can be mapped to MuxApplication under the assumption that bytesCS and bytesBF coincide. Using different bytes for different application is useful for running different encoding on each channel in tests (identity codecs).
