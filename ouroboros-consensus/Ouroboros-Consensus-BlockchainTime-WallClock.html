Wait until the next slot

Takes the current slot number to guard against system clock changes. Any clock changes that would result in the slot number to decrease will result in a fatal SystemClockMovedBackException. When this exception is thrown, the node will shut down, and should be restarted with (full?) validation enabled: it is conceivable that blocks got moved to the immutable DB that, due to the clock change, should not be considered immutable anymore.
