data TPraosState c Source #

Praos consensus state.

The transitional praos implementation itself has a concept of state, given by the STS.State (PRTCL c) type. This state, however, doesn't support rewinding.

Rewind the state to the specified slot

The state is rewound to its position at the _end_ of the specified slot (i.e. after any blocks in that slot have been applied)

Callers of this function should ensure that the slot we are requesting to rewind to contains a block which was previously applies. However, we do not technically require this - if a block has not been applied in this slot, then we simply return the state as it was following the last applied block.
