ForgeState

data TPraosForgeState c Source #

This is the state containing the private key corresponding to the public key that Praos uses to verify headers. That is why we call it TPraosForgeState instead of ShelleyForgeState.

Constructors

TPraosKeyAvailable !(HotKey c)

The online KES key used to sign blocks is available at the given period

TPraosKeyEvolving

The KES key is being evolved by another thread

Any thread that sees this value should back off and retry.

TPraosNoKey

This node is not a core node, it doesn't have the capability to sign blocks.

The ForgeState of such a node will always be ShelleyNoKey.
